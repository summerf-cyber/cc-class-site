## Project: Procurable.ai Marketing Site

### Goals
- Generate qualified leads from VP Procurement / CPO / Director of Strategic Sourcing at aerospace and advanced manufacturing companies ($500M–$5B revenue)
- Communicate Procurable.ai's should-cost intelligence value proposition in the buyer's own language
- Drive pipeline through two CTAs: ROI calculator (top of funnel) and pilot request (bottom of funnel)

### Stack
- Plain HTML, CSS, and JavaScript — no frameworks, no build step
- Single file: `index.html`
- Deployed on Vercel

### Copy Rules
- **Always reference `persona.md` before writing or editing any copy** — every headline, subhead, CTA, and body paragraph should speak to Marcus Chen's specific pain points, language, and skepticism
- Use procurement-native terminology: should-cost, TCO, NPI, cost avoidance, commodity volatility, audit-ready
- Lead with outcomes and hard numbers, not features
- Address skepticism directly — Marcus has been burned by enterprise software promises before
- Never use vague "AI-powered" language without a specific, defensible claim behind it

### Key Files
- `index.html` — full marketing site including the ROI calculator modal
- `persona.md` — target buyer profile (read before touching any copy)
- `prd.md` — ROI calculator product requirements
